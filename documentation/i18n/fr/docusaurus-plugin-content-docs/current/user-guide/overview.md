# Vue d'ensemble

Ce guide vous accompagne dans Invoicerr écran par écran. Chaque page ci-dessous correspond à un écran de l'application.

## Connexion

Ouvrez votre instance Invoicerr et connectez-vous depuis la page **Sign in** avec votre adresse e-mail et mot de passe. Si votre instance utilise l'authentification unique (OIDC), utilisez le bouton SSO à la place.

## Premier démarrage

La première fois que vous ouvrez l'application, si le nom de votre entreprise n'est pas encore défini, Invoicerr ouvre automatiquement une boîte de dialogue qui vous mène dans **Settings → Company**. Remplissez-la avant toute autre chose : le nom, l'adresse, la devise et la numérotation de votre entreprise apparaissent sur chaque devis, facture et reçu émis.

Un ordre recommandé pour bien démarrer :

1. **[Settings → Company](settings.md#company)** — votre identité commerciale et la numérotation
2. **[Clients](clients.md)** — les personnes et entreprises que vous facturez
3. **[Moyens de paiement](payment-methods.md)** — comment vos clients vous paient
4. **[Devis](quotes.md)** → **[Signature de devis](quote-signing.md)** → **[Factures](invoices.md)** → **[Reçus](receipts.md)** — le flux documentaire

## La barre latérale

Une fois connecté, la barre latérale gauche est votre navigation principale. Les éléments apparaissent dans cet ordre :

| Élément | Utilité |
| --- | --- |
| **Dashboard** | Aperçu des revenus, devis, factures et clients |
| **Quotes** | Créer des devis et les envoyer pour signature |
| **Invoices** | Facturer les clients et suivre les paiements |
| **Receipts** | Émettre des preuves de paiement |
| **Clients** | Gérer les personnes et entreprises facturées |
| **Payment Methods** | Options de paiement réutilisables sur les documents |
| **Statistics** | Analyses de revenus dans le temps |
| **Settings** | Identité de l'entreprise, modèles, intégrations et compte |

Le pied de la barre latérale contient le sélecteur de thème (Clair / Sombre / Système) et votre menu de profil (Compte, Déconnexion).

Prêt ? Commencez par les [Clients](clients.md).
